typedef BaseComponentId<uint64_t, uint32_t, 32, uint32_t, 24, uint8_t, 8> Kunlaboro::ComponentId

The default component ID type.

Note
32 bits for index, meaning ~4.3 billion components possible.
24 bits for generation, meaning ~16.7 million component slots can be reused before component IDs become non-unique.
8 bits for family, meaning 256 distinct component types can exist.
