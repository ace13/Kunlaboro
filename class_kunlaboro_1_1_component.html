Detailed Description

The Kunlaboro Component base class.

This is the base class that needs to be used when registering component types for use in the Kunlaboro entity system.

Simple example dealing with a cooldown for something;

class CooldownComponent : public Kunlaboro::Component
{
public:
CooldownComponent(float time)
: Time(time)
{ }
void update(float dt)
{
Time -= dt;
if (Time > 0)
return;
getEntitySystem()->destroyComponent(getId());
}
// Time remaining on the cooldown
float Time;
};
// Usage example
void GameLogic::update(float dt)
{
// ...
if (ability.hasComponent<CooldownComponent>())
ability.getComponent<CooldownComponent>()->update(dt);
else
// ...
}

Member Function Documentation

EntityId Kunlaboro::Component::getEntityId ( ) const

Gets the ID of the entity containing the component.

Each component can only be held by a single entity at a time, so this is guaranteed to be either a valid entity or EntityId::Invalid()

Note
This function will require iterating the entity table, so performance is O(n) on number of existing entities. Recommended is to cache the resulting value if it's to be used repeatedly.

const ComponentId& Kunlaboro::Component::getId ( ) const

Gets the ID of the component.

On creation each component is given an identifier based on its type, index, and generation of that index. This is guaranteed to be unique in most cases, while still favouring lower indexes and therefore faster memory lookups.

Note
This identifier may no longer be unique after ComponentId::sMaxGeneration worth of generations have passed.
